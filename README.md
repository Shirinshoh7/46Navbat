
**Клиент-серверное решение для автоматизации распределения потока посетителей. Система построена на базе микросервисного подхода (Web API) и обеспечивает передачу данных в реальном времени.**



* **Backend:** .NET 9 (ASP.NET Core Web API)
* **Database:** PostgreSQL (Развертывание через Docker)
* **ORM:** Entity Framework Core (Code First)
* **Real-time Communication:** SignalR (WebSockets)
* **Frontend:** HTML5, CSS3, JavaScript (Vanilla JS)



Проект разделен на три независимых функциональных модуля, которые взаимодействуют через единый API:

1.  **Административная панель (Admin Panel):**
    * Конфигурация категорий и перечня услуг.
    * Управление жизненным циклом заявки (вызов клиента, смена статуса, завершение обслуживания).
    * Мониторинг текущей нагрузки на операторов.

2.  **Терминал самообслуживания (Client Terminal):**
    * Интерфейс выбора услуги для клиента.
    * Генерация уникального идентификатора талона (префикс услуги + порядковый номер) с сохранением в БД.

3.  **Информационное табло (Monitor Display):**
    * Мгновенное обновление данных через SignalR при вызове нового клиента.
    * Интеграция с Browser Speech API для голосового оповещения посетителей.



База данных спроектирована с учетом нормализации и включает следующие сущности:
* **Category:** Группировка направлений деятельности.
* **Service:** Справочник услуг с привязкой префиксов для нумерации.
* **Ticket:** Транзакционная таблица (регистрация времени создания, номера талона и текущего статуса: waiting, calling, completed).



Для запуска системы в локальном окружении необходимо выполнить следующие шаги:


Запустите контейнер PostgreSQL:
```bash
docker run --name navbat-db -e POSTGRES_PASSWORD=your_password -p 5434:5432 -d postgres
```

Обновите структуру БД через EF Core:
```bash
dotnet ef database update
```


Запустите серверную часть:
```bash
dotnet run
```
Клиентские интерфейсы доступны в директории `wwwroot` (`admin.html`, `terminal.html`, `monitor.html`).

* Внедрение аутентификации на базе JWT для разграничения доступа операторов.
* Модуль аналитики: расчет среднего времени ожидания и обслуживания для оптимизации работы персонала.
* Разработка мобильного интерфейса для удаленного бронирования очереди.
